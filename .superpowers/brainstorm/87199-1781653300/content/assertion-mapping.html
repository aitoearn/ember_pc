断言如何在自然语言用例中体现

执行引擎是 VLM ReAct（模型看屏幕自己操作+判断），所以断言 = 写进 instruction 的「通过条件」，由模型自判。下面是映射关系。

结构化用例（库里存的）
TC-LOGIN-001 · 登录成功
前置：已安装App、未登录
步骤1 输入正确账号密码并点登录
　预期：进入首页
步骤2 查看首页顶部
　预期：显示「欢迎，张三」
整体预期：成功登录并停留在首页
转成的 instruction（喂给模型）
【任务】完成登录并验证
【前置】已安装App、未登录
【操作步骤】
1. 输入账号密码并点击登录
2. 查看首页顶部
【通过条件 / 断言】
✓ 登录后进入首页
✓ 首页顶部显示「欢迎，张三」
只有全部满足才报成功；
若某条不满足，指出是哪条。
模型自判 → 结果
模型执行中逐步对照「通过条件」，结束时自报：success=true/false + finalMessage（哪条没过）→ 回写用例 execResult。

软断言能做到

  • 每步 expected、整体预期都进 prompt
  • 模型对照判断、报出失败原因
  • 可用 prompt 结构强化（必须逐条核对）

现状做不到（属未来 App Server 路线）

  • 确定性逐步断言 / 截图比对 / DOM 断言
  • 模型可能误判（幻觉）
  • 不保证「显示欢迎张三」像素级正确

# 自动研究协议（program）

> `obsidian-autoresearch` 的完整执行协议。本文件被技能在执行 Deep 研究时按需读取。

## 研究循环（research loop）

```
INPUT: 1–3 个研究问题 + 现有知识库覆盖快照
循环（最多 3 轮）：
  1. 生成检索查询（每问 2–3 个变体）
  2. WebSearch → 候选源列表
  3. 质量过滤：一手来源 > 二手；近期 > 陈旧；权威域名优先
  4. WebFetch 抓取 top 候选（单轮 ≤ 5）
  5. 抽取与现有知识的差异（新增/印证/矛盾）
  6. 判断是否已回答研究问题：
     - 是 → 退出循环
     - 否 → 用剩余空白生成下一轮查询
OUTPUT: 入库的源 + 新建/更新的 wiki 页 + 残留空白清单
```

## 源质量评分

| 维度 | 高分 | 低分 |
| --- | --- | --- |
| 来源层级 | 一手（论文、官方文档、原作者） | 三手聚合 |
| 时效 | 近期 | 明显过时 |
| 可验证 | 有出处、可复核 | 无引用 |

## 矛盾优先

研究中发现的矛盾是最高信号。务必用 `> [!contradiction]` 在相关页双向标注，并在 `questions/` 立一条待解问题。

## 边界与停止条件

- 单轮源数 ≤ 5；总轮次 ≤ 3。
- 连续两轮无新增信息 → 停止，报告残留空白。
- 任何阻塞（登录/付费/验证码）→ 停止并报告，不绕行。
